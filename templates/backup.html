📤 Restore from Backup Admin Only

Upload a previously downloaded backup file to restore data. Supports both JSON and Excel (.xlsx) backup files. Choose Merge to add missing records without touching existing ones, or Full Replace to wipe documents and reload everything fresh.

⚠️

Full Replace permanently deletes all current documents and routing slips before restoring. Users are never deleted for safety. Always download a fresh backup before doing a Full Replace.

ℹ️ How to Use Backup & Restore
  • 1
    Download regularly — click "Download as Excel" weekly or before any major changes. The Excel file is great for records and printing. Save with a date in the filename.
  • 2
    Also keep a JSON backup — the .json file is what you upload to restore data back into DocTracker. Keep both formats.
  • 3
    Store safely — save to your PC, a USB drive, or Google Drive. Keep multiple dated copies.
  • 4
    After data loss — if the database resets (Railway free tier restarts, new deploy, etc.), use Restore with your .json backup file to reload everything in seconds.
  • 5
    Use Merge mode for most restores — it only adds missing records and never overwrites what's already there.
  • 6
    Use Full Replace only when you want to roll back the entire database to a known good state. Download a fresh JSON backup first.
